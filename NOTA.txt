#  IMPORTANTE – SISTEMA CLIENTE / SERVIDOR (LEER COMPLETO)

## 🧠 REGLA PRINCIPAL
El cliente NUNCA usa `localhost` cuando se conecta a otra máquina. Siempre debe usar la IP del servidor.

## 🌐 FUNCIONAMIENTO GENERAL
1. Se ejecuta el servidor en una máquina (Windows, Linux o Mac)
2. El servidor escucha en el puerto 8888
3. El cliente en otra máquina se conecta usando la IP del servidor

## 🐧 SI EL SERVIDOR ESTÁ EN LINUX (VMWARE)

En Linux ejecuta esto antes del servidor:
sudo ufw allow 8888

Servidor:
python3 run_server.py

Obtener IP:
ip a

Ejemplo IP:
192.168.73.128

Cliente (Windows/Mac):
python run_client.py 192.168.73.128 8888

## 🪟 SI EL SERVIDOR ESTÁ EN WINDOWS
Servidor:
python run_server.py

Obtener IP:
ipconfig

Cliente (Linux/Mac):
python3 run_client.py IP_DE_WINDOWS 8888

## 🍎 SI EL SERVIDOR ESTÁ EN MAC
Servidor:
python3 run_server.py

Obtener IP:
ip a

Cliente (Windows/Linux):
python run_client.py IP_DE_MAC 8888

## ⚠️ ERRORES COMUNES
- localhost = solo funciona en la misma máquina
- Servidor apagado = WinError 10061
- IP incorrecta = no conecta
- Firewall bloqueando puerto 8888
- VMware en NAT = puede bloquear conexión entre máquinas

## 🔎 VERIFICACIÓN
ping IP_DEL_SERVIDOR

Si responde = conexión OK

## 🚀 CONFIGURACIÓN RECOMENDADA
- Servidor encendido antes del cliente
- Cliente usando IP real del servidor
- Puerto 8888 abierto
- VMware en BRIDGED recomendado
- Verificar IP antes de conectar

## 🧩 RESUMEN FINAL
Servidor define la IP, cliente usa esa IP, nunca localhost entre máquinas, primero servidor luego cliente, si falla revisar IP, red o firewall
